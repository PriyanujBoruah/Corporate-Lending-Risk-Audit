3.2. Financial Sanity Check: Loan Installment Calculation

The monthly installment is a function of loan_amount, interest_rate, and loan_term. While the exact formula is complex, large deviations from an estimated calculation can flag incorrect loan data.

In[]:
# A simplified check for loan installment. NOTE: This is an approximation.
# Real formula is complex. This is for flagging major errors.
monthly_rate = (df['interest_rate'] / 100) / 12

features = [

    'loan_amount', 'interest_rate', 'loan_term', 'num_of_open_accounts',
    'credit_utilization', 'income_per_capita', # Our engineered features!

9. State-of-the-Art Model: XGBoost (Extreme Gradient Boosting)

Objective: When maximum accuracy is the goal, XGBoost is the industry standard for tabular data. It's a highly sophisticated ensemble model that learns from its mistakes sequentially.

Key Parameter: We will use scale_pos_weight. This is the XGBoost way of handling imbalanced classes, similar to class_weight in Random Forest. We calculate the ratio of non-defaulters to defaulters to tell the model how much more important the 'Default' class is.

Cell 1: Training the XGBoost Model

import xgboost as xgb

# --- Parameters for Adjusting ---
# n_estimators: Number of boosting rounds (trees).
# learning_rate (eta): A small number (e.g., 0.05-0.3). It shrinks the contribution of each new tree, preventing overfitting.
# max_depth: The maximum depth of each tree.
# scale_pos_weight: THE KEY for imbalanced data. Tells the model to heavily penalize misclassifying the minority class (Defaults).
ratio = (y_train == 0).sum() / (y_train == 1).sum()

xgb_model = xgb.XGBClassifier(
    n_estimators=25000,
    learning_rate=0.035,
    max_depth=5,
    scale_pos_weight=ratio,      # Crucial for imbalanced data
    use_label_encoder=False,
    eval_metric='logloss',
    random_state=42
)

# Train the model
print("Training XGBoost model...")
xgb_model.fit(X_train, y_train)
print("Training complete.")
